## Алгоритмы и структуры данных (критично для собесов)

### Зачем нужно
- 90% технических собеседований включают алгоритмы
- Критично для оптимизации кода ОС
- Используется в планировщиках, файловых системах, memory management

### Что изучить

#### 1. Алгоритмы сортировки
- **O(n²):** Bubble Sort, Selection Sort, Insertion Sort
- **O(n log n):** Merge Sort, Quick Sort, Heap Sort
- **Специальные:** Counting Sort, Radix Sort

#### 2. Big O notation (алгоритмическая сложность)
```
O(1)        - Константная
O(log n)    - Логарифмическая (бинарный поиск)
O(n)        - Линейная
O(n log n)  - Линейно-логарифмическая (быстрые сортировки)
O(n²)       - Квадратичная
O(2^n)      - Экспоненциальная
```

#### 3. Структуры данных
- **Linked Lists:** Односвязные, двусвязные
- **Stack & Queue:** LIFO, FIFO
- **Trees:** Binary Tree, BST, AVL, Red-Black Tree
- **Hash Tables:** Hash functions, collision resolution
- **Heap:** Min-heap, Max-heap, Priority Queue
- **Graphs:** BFS, DFS (базовые алгоритмы)

## Важные правила

### ДЕЛАТЬ:
- Писать код каждый день (даже 30 минут)
- Пробовать самому перед тем как смотреть решение
- Экспериментировать с кодом
- Записывать заметки своими словами
- Коммитить часто (каждый день)
- Компилировать с -Wall -Wextra -Werror всегда
- Проверять на утечки с valgrind
- Задавать вопросы и искать ответы, когда непонятно

### НЕ ДЕЛАТЬ:
- Копировать код не понимая
- Пропускать упражнения

- Торопиться (качество > скорость)
- Игнорировать warnings компилятора
- Забывать free() после malloc()
- Стесняться задавать вопросы

## Полезные команды

### Компиляция
```bash
# Базовая компиляция
gcc program.c -o program

# С предупреждениями (рекомендуется всегда)
gcc -Wall -Wextra -std=c11 program.c -o program

# С отладочной информацией
gcc -g -Wall -Wextra program.c -o program

# Со строгими правилами (warnings = errors)
gcc -Wall -Wextra -Werror program.c -o program
```

### Отладка
```bash
# Valgrind (проверка утечек памяти)
valgrind --leak-check=full ./program

# GDB (отладчик)
gdb ./program
# В GDB:
# break main    - точка останова
# run          - запуск
# next         - следующая строка
# print var    - вывести переменную
# backtrace    - стек вызовов
```

### Git
```bash
# Ежедневный workflow
git status
git add .
git commit -m "описание изменений"
git push

# Просмотр истории
git log --oneline
git log --graph --oneline

# Откат изменений
git checkout -- file.c        # отменить изменения
git reset HEAD file.c         # убрать из stage
```
